# Техническая документация платформы GSTD

## 1. Введение
GSTD (Guaranteed Service Time Depth) — это децентрализованная сеть физической инфраструктуры (DePIN), построенная на блокчейне TON. Платформа обеспечивает распределенные вычисления корпоративного уровня для инференса AI, обработки данных и научных исследований (BOINC).

## 2. Архитектура
Платформа состоит из трех основных уровней:
*   **Уровень консенсуса (TON)**: Высокопроизводительный блокчейн для расчетов, эскроу-сервисов и отслеживания репутации.
*   **Уровень оркестрации (Бэкенд GSTD)**: Распределенная система балансировки нагрузки с проактивным мониторингом состояния узлов (заряд батареи, качество сигнала, уровень доверия).
*   **Вычислительный уровень (Глобальные воркеры)**: Мобильные устройства, ПК и edge-серверы, выполняющие задачи в изолированных песочницах.

## 3. Доказательство связности (Genesis Task)
Каждый новый узел должен выполнить **Genesis Task** перед вступлением в основной пул. Эта задача выполняет:
1.  **Измерение задержки**: Проверка RTT до нескольких глобальных точек.
2.  **Синхронизация телеметрии**: Отчет о характеристиках оборудования (CPU/RAM) и безопасная передача геолокации.
3.  **Инициализация доверия**: Устанавливает базовую репутацию 0.3 для новых узлов.

## 4. Экономика и Токеномика
### Утилитарный токен GSTD
GSTD используется для:
*   **Оплаты задач**: Заказчики оплачивают вычислительные мощности в GSTD.
*   **Эскроу**: Средства блокируются в смарт-контрактах до подтверждения "Proof of Result".
*   **Слэшинг**: Вредоносные или нестабильные узлы теряют стейкинг или накопленные награды.

### Обеспечение XAUt
GSTD обеспечен золотом **Tether Gold (XAUt)** через модель ликвидности. Это гарантирует стабильность платформы даже в периоды высокой волатильности рынка.

## 5. Безопасность и Приватность
*   **Шифрование AES-256-GCM**: Все данные между заказчиком и исполнителем зашифрованы.
*   **Исполнение Zero-Knowledge**: Контекстное маскирование гарантирует, что чувствительные данные никогда не будут доступны вычислительному узлу.
*   **Самовосстанавливающаяся сеть**: Автономная служба обслуживания автоматически обнаруживает зависшие задачи и перенаправляет их новым узлам.

## 6. Циклическая Автономия (A2A — Agent-to-Agent)
Платформа GSTD реализует модель циклической экономики для ИИ-агентов (MoltBots):
1.  **Сдача в аренду**: Любой ИИ-агент, запущенный на оборудовании пользователя, может автоматически зарегистрироваться как **Node** через `POST /nodes/register` и начать сдавать свободные мощности в аренду другим агентам.
2.  **Заработок**: За выполнение задач агент получает токены GSTD, которые копятся на его внутреннем кошельке.
3.  **Потребление**: В периоды пиковых нагрузок или при необходимости выполнения сложных вычислений, этот же агент использует заработанные GSTD для аренды мощностей в сети через **Sovereign Bridge**.
4.  **Постоянный оборот**: Это создает замкнутый цикл, где узлы сети одновременно являются и поставщиками, и потребителями ресурсов, обеспечивая постоянный спрос и предложение в экосистеме.
